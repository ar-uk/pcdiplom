# Parts Blueprint

This blueprint defines the tables to support per-part CSV imports.

## Logical part to table mapping

- CPU -> cpu
- GPU -> video_card
- POWER_SUPPLY -> power_supply
- PC_CASE -> pc_case
- RAM -> memory
- INTERNAL_MEMORY -> internal_hard_drive
- MOTHERBOARD -> motherboard

## Recommended CSV headers by part

### CPU (table: cpu)
name,price_usd,core_count,core_clock,boost_clock,microarchitecture,tdp,graphics

### GPU (table: video_card)
name,price_usd,chipset,memory_gb,core_clock,boost_clock,color,length_mm

### POWER_SUPPLY (table: power_supply)
name,price_usd,psu_type,efficiency,wattage,modular,color

### PC_CASE (table: pc_case)
name,price_usd,case_type,color,psu,side_panel,external_volume,internal_35_bays

### RAM (table: memory)
name,price_usd,ddr,speed_mhz,sticks,gb_per_stick,price_per_gb,color,first_word_latency,cas_latency

### INTERNAL_MEMORY (table: internal_hard_drive)
name,price_usd,capacity_gb,price_per_gb,drive_type,cache_mb,form_factor,interface

### MOTHERBOARD (table: motherboard)
name,price_usd,socket,form_factor,max_memory,memory_slots,color

## Notes

- id is auto-generated by the database for all tables.
- name is required in all part tables.
- The migration is idempotent (CREATE TABLE IF NOT EXISTS).
- LOWER(name) indexes are added to speed up case-insensitive name matching.
